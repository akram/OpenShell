<Warning>
The OpenShift install path is experimental. It currently requires running sandbox pods under the `privileged` SCC. Use only for evaluation on a private network.
</Warning>

Deploy the OpenShell gateway on OpenShift with Security Context Constraints, external access via Routes or Gateway API, and OIDC authentication with Keycloak.

<Cards>

<Card title="Install" href="/kubernetes/openshift/install">

Install the gateway with TLS enabled, configure the SCC binding, and verify the deployment.
</Card>

<Card title="Gateway Connection" href="/kubernetes/openshift/gateway-connection">

Connect the CLI to the gateway using port forwarding, a reencrypt Route, or the Gateway API.
</Card>

<Card title="OIDC with Keycloak" href="/kubernetes/openshift/oidc-keycloak">

Configure Keycloak as the OIDC provider, set up the required protocol mappers, and connect the CLI.
</Card>

<Card title="OpenShift Identity Federation" href="/kubernetes/openshift/identity-federation">

Let cluster users log in with their OpenShift credentials via Keycloak identity federation.
</Card>

</Cards>
